# CAPA Procedure

## 1. Purpose

This procedure defines the process for identifying, investigating, and resolving corrective and preventive actions (CAPA) to eliminate the causes of actual or potential nonconformities and prevent their recurrence or occurrence.

**Related documents:** [[SOP-001]] Document Control, [[FM-001]] CAPA Form, [[DWG-001]] CAPA Process Flow

## 2. Scope

This procedure applies to all nonconformities identified through:
- Customer complaints
- Internal and external audits
- Process monitoring and measurement
- Post-market surveillance
- Management review
- Regulatory reporting

## 3. Responsibilities

| Role | Responsibility |
|------|---------------|
| Quality Manager | Oversees CAPA process, approves closures, reports to management |
| Process Owner | Investigates root cause, defines and implements corrective/preventive actions |
| CAPA Initiator | Identifies nonconformity, opens CAPA record |
| Management Representative | Reviews CAPA trends, ensures resources |

## 4. Procedure

### 4.1 Identification and Documentation

When a nonconformity or potential nonconformity is identified:

1. Open a CAPA record using [[FM-001]]
2. Assign a unique CAPA number
3. Document the source, description, and initial severity assessment
4. Notify the Quality Manager within 24 hours for High/Critical severity

### 4.2 Severity Classification

| Severity | Criteria | Timeline |
|----------|----------|----------|
| Critical | Patient safety risk, regulatory non-compliance | Investigation within 24h |
| High | Significant quality impact, recurring issue | Investigation within 5 days |
| Medium | Minor quality impact, isolated incident | Investigation within 15 days |
| Low | Observation, improvement opportunity | Investigation within 30 days |

### 4.3 Root Cause Analysis

The Process Owner shall:
1. Gather relevant data and evidence
2. Conduct root cause analysis using appropriate methods (5 Why, Fishbone, etc.)
3. Document the root cause in the CAPA record
4. Identify whether the root cause could affect other processes or products

### 4.4 Corrective and Preventive Actions

Based on the root cause analysis:
1. Define corrective actions to address the immediate nonconformity
2. Define preventive actions to prevent recurrence
3. Assess the impact of proposed actions on product safety and performance
4. Set target completion dates
5. Obtain approval from Quality Manager before implementation

### 4.5 Implementation

1. Implement approved actions according to the defined timeline
2. Update affected documents as needed per [[SOP-001]]
3. Provide training if procedures are changed
4. Document implementation evidence

### 4.6 Effectiveness Verification

After implementation:
1. Define effectiveness verification criteria
2. Allow sufficient time for verification (minimum 30 days for process changes)
3. Collect and analyze data to confirm effectiveness
4. If not effective, return to root cause analysis (4.3)

### 4.7 Closure

1. Quality Manager reviews CAPA record for completeness
2. Verify all actions implemented and effective
3. Close CAPA record
4. Update risk management file if applicable ([[RA-001]])
