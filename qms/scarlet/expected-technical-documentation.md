# Application Review: Expected Technical Documentation

At the application review stage, Scarlet will assess your submitted technical documentation set for completeness, not conformity.

When preparing your documentation for submission, it is recommended that you strive for completeness and quality rather than perfection. Further revision of the documentation is possible during the conformity assessment process. The iterative rounds of assessment, queries and resolution can be effective in addressing minor compliance gaps within your documentation.

**Caveat:** The number of rounds for the queries is limited to preserve impartiality, so they should be used wisely.

The set of documents provided here is split into core documentation, which is required for application review, and supporting documentation, which may be requested during conformity assessment.

## Expected Supporting Documentation

The following supporting technical documentation may also be required for conformity assessment. It is considered non-blocking for application review, provided it can be made available to Scarlet upon request during the conformity assessment.

As stated in EU MDR, Annex IX, 2.2:

> "In addition, the manufacturer shall grant the notified body access to the technical documentation referred to in Annexes II and III."

If not initially provided in your application, Scarlet may request further information during the conformity assessment process related to some of the following documentation.

### Submission Context

- Previous generations and similar devices

### Non-clinical Evidence

### Software

- Cybersecurity threat modelling
- Cybersecurity testing
- Software traceability matrix

### Usability Engineering

- Formative evaluation plans
- Formative evaluation reports
- Usability traceability matrix

### Clinical Evidence

- Clinical investigator's brochure(s)
- Summary of safety and clinical performance (for Class III devices)

### Labelling and Accompanying Documentation

- Other documentation, such as:
  - Technical and/or operator manuals
  - User training documentation
  - e-IfU
