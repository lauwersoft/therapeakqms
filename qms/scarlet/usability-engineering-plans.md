# Usability Engineering Plans

> Documentation that defines the usability engineering formative and summative evaluation activities planned to ensure the user interface can be used safely and effectively.

Usability engineering plans define both formative and summative evaluation activities planned as part of the medical device software development process. Formative evaluation activities are performed iteratively during development to identify and address usability issues early, while summative evaluation activities are performed on the final design to obtain objective evidence that the user interface can be used safely.

Regulatory expectations under IEC 62366-1 and EU MDR require manufacturers to plan comprehensive usability engineering activities that address hazard-related use scenarios, use errors, and risk control measures defined to mitigate risks arising from use errors. These plans are essential to demonstrate that usability risks have been systematically identified and addressed throughout the development lifecycle and that the medical device's user interfaces are safe to use.

## Minimum Requirements

When submitting your technical documentation, the following are the minimum requirements for this topic:

- **Responsible personnel**: The plan must identify personnel responsible for usability engineering evaluation activities and provide evidence of their competency for these activities.
- **Scope definition**: The plan must describe the sections of the user interface(s) and user groups in scope, and identify the hazard-related use scenarios, use errors, and use requirements (and for summative evaluations, risk control measures) that are being evaluated.
- **Evaluation activities**: The plan must define a set of formative and summative evaluation activities, each including a description of the activity objective and procedure, the methods proposed, the scope of user interface sections and user groups, and the hazard-related elements being evaluated.
- **Summative evaluation specifics**: For planned summative evaluation activities, more comprehensive details are expected. This includes justification for scope selection, definition of participant numbers and selection criteria, justification for user groups and inclusion/exclusion criteria, and details on the accompanying documentation and user training required to facilitate the execution of the activity.

> **Note:** For EU MDR certification, formative evaluation activities are not specifically required for certification but are strongly recommended as best practice. Formative evaluation activities conducted throughout the development lifecycle can help identify and address usability issues early, reducing the likelihood of safety issues in the final product.

See the [deep dive section](#deep-dive) for more details on the expected content of these usability engineering plans, including formative and summative evaluation activities.

## Common Pitfalls

The following are common queries that are raised by Scarlet during assessment:

- **Missing or inadequate personnel competency evidence**: Plans that fail to identify responsible personnel or provide insufficient evidence of their competency for usability engineering evaluation activities.
- **Incomplete scope definition**: Plans that do not clearly describe the sections of user interface(s) and user groups in scope, or that fail to identify the relevant hazard-related use scenarios, use errors, or risk control measures that are being evaluated.
- **Insufficient evaluation activity details**: Evaluation activities that lack essential information such as clear objectives, procedures, methods, selection criteria, or accompanying documentation.

## Desired Submission Format

Scarlet specifies desired submission formats for certain technical documentation to facilitate easier assessment. It is recommended that usability engineering plans are provided as a standard document format, such as .DOCX or .PDF.

## Deep Dive

### Expected Content: Formative Evaluation Plans

#### Responsible Personnel

Include:

- The identification of the personnel responsible for the usability engineering formative evaluation plan and evidence of their competency for this activity.

#### Scope

Include:

- **User interface sections and user groups**: A description of the sections of the user interface(s) and user groups in the scope of the formative evaluation.
- **Usability risk elements**: The selection of hazard-related use scenarios, use errors, risk control measures and use requirements which are in the scope of the formative evaluation. Note: These will have been identified through risk analysis and requirements analysis activities.

#### Formative Evaluation Activities

Include a set of usability engineering formative evaluation activities. Each activity should include the following information:

- A high-level description of the activity, including its objective and procedure.
- A description of the methods proposed for the activity. The methods may include an expert review, a review of relevant standards, heuristic analysis, or cognitive walkthroughs.
- A description of the sections of the user interface(s) and user groups within the scope of the activity.
- Indicate the hazard-related use scenarios, use errors and use requirements that the activity evaluates.

### Expected Content: Summative Evaluation Plans

#### Responsible Personnel

Include:

- The identification of the personnel responsible for the usability engineering summative evaluation plan and evidence of their competency for this activity.

#### Scope

Include:

- **User interface sections and user groups**: A description of the sections of the user interface(s) and user groups in the scope of the summative evaluation.
- **Usability risk elements**: The selection of hazard-related use scenarios, use errors, risk control measures and use requirements which are in the scope of the summative evaluation. Note: These will have been identified through risk analysis and requirements analysis activities.
- **Scope justification**: A justification for this scope selection, if not all user interfaces, groups and usability risk elements are being evaluated.

#### Summative Evaluation Activities

Include a set of usability engineering summative evaluation activities. Each activity should include the following information:

- A high-level description of the activity, including its objective and procedure.
- A description of the methods proposed for the activity and justify that this method is adequate to produce the required objective evidence. The methods may include an expert review, a review of relevant standards, heuristic analysis, or cognitive walkthroughs.
- A description of the sections of the user interface(s) and user groups within the scope of the activity.
- Indicate the hazard-related use scenarios, use errors, risk-control measures, and use requirements that the activity evaluates.
- A definition of the number of participants, the user groups, and the use environments selected for the activity.
- A justification of the selected user group and any defined inclusion and exclusion criteria.
- A definition of the selection criteria for personnel performing the activity, and if applicable, include the requirements for an expert review.
- A description of any accompanying documentation and/or user training required to facilitate the execution of the activity.
- Any specific details about the provisioning of user training ahead of the activity execution.
